Langkah Setup GitHub Pages

1. Repository Setup

Pastikan repository Anda bernama username.github.io atau memiliki branch gh-pages.

2. Konfigurasi Pages

Buka Settings > Pages, pilih source (branch atau GitHub Actions).

3. Custom Domain

Tambahkan domain custom dan konfigurasi DNS records di provider domain Anda.

Konfigurasi GitHub Actions

Untuk website SempoaAR, Anda memerlukan file workflow GitHub Actions. Berikut contohnya:

# File: .github/workflows/jekyll.yml
name: Deploy Jekyll site to Pages

on:
  push:
    branches: ["main"]
  pull_request:

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v3
      - name: Setup Pages
        uses: actions/configure-pages@v2
      - name: Build with Jekyll
        uses: actions/jekyll-build-pages@v1
        with:
          source: ./
          destination: ./_site
      - name: Upload artifact
        uses: actions/upload-pages-artifact@v1

  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    needs: build
    steps:
      - name: Deploy to GitHub Pages
        uses: actions/deploy-pages@v1

Pemecahan Masalah (Troubleshooting)

Domain tidak dapat diakses

Jika domain GitHub Pages Anda (username.github.io) tidak dapat diakses, kemungkinan penyebabnya:

  • Repository tidak bernama username.github.io (untuk user pages)
  • Branch gh-pages tidak ada (untuk project pages)
  • GitHub Actions workflow gagal
  • File index.html tidak ada di root directory

Custom domain tidak bekerja

Jika Anda menambahkan custom domain tetapi tidak berfungsi:

  1. Pastikan Anda telah menambahkan domain di pengaturan GitHub Pages
  2. Periksa DNS records di provider domain Anda:
    • Untuk apex domain (contoh.com): A records menuju 185.199.108.153, 185.199.109.153, 185.199.110.153, 185.199.111.153
    • Untuk subdomain (www.contoh.com): CNAME record menuju username.github.io
  3. Tunggu hingga 24 jam untuk propagasi DNS
  4. Pastikan file CNAME (jika digunakan) berisi domain Anda

HTTPS tidak aktif

GitHub Pages menyediakan HTTPS gratis untuk custom domain. Jika tidak aktif:

  • Pastikan DNS sudah terpropagasi sempurna
  • Hapus dan tambahkan kembali custom domain di pengaturan
  • Pastikan tidak ada konten campuran (HTTP dan HTTPS) di website Anda

Hubungi Dukungan GitHub

Konfigurasi Custom Domain

Langkah Konfigurasi Domain
  1. Beli domain dari provider (GoDaddy, Namecheap, dll)
  2. Di repository GitHub, buka Settings > Pages
  3. Di bagian "Custom domain", masukkan domain Anda
  4. Klik Save
  5. Login ke provider domain Anda
  6. Tambahkan DNS records sesuai petunjuk
  7. Tunggu hingga 24 jam untuk propagasi DNS
  8. Centang opsi "Enforce HTTPS"
Contoh DNS Configuration

Untuk domain example.com:

A Records (untuk apex domain):
Type Name Value
A @ 185.199.108.153
A @ 185.199.109.153
A @ 185.199.110.153
A @ 185.199.111.153
CNAME Record (untuk www subdomain):
Type Name Value
CNAME www username.github.io
